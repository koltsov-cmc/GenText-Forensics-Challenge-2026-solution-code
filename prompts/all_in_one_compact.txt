==================================================================
ABOUT DTD
==================================================================

DTD achieves 0.66 IoU vs ground-truth — when it flags a region, that region is GENERALLY real tampering. Default = KEEP every flagged region.

DTD has 3 blind spots:
1. DTD does NOT classify the type — YOU read the content and decide Visual / Logical / Semantic.
2. DTD misses purely-semantic substitutions (clean word → clean word). YOU must find these yourself.
3. DTD misfires on math formulas, dense tables, logos/stamps/seals.

==================================================================
KEEP RULES (default — apply when classifying TYPE)
==================================================================

A. Region inside paragraph/between words → Visual Clumsy if visible artefact, Semantic Subtle if clean word substitution, Logical Fraud if numeric value.
B. Blurred / JPEG-distorted text → Visual Clumsy.
C. Solid rectangle / white-out / black-out → Visual Clumsy.
D. Empty area in middle of running text → Visual Clumsy or Logical Fraud (removed value).
E. Blurred logo / stamp / seal in otherwise sharp doc → Visual Clumsy.
F. Numeric value, digits clean (number/date/percentage/total/year) → Logical Fraud.
G. Clean word/phrase in prose contradicting surrounding context → Semantic Subtle.

==================================================================
DROP RULES (only valid reasons to remove a DTD region)
==================================================================

H. Tiny, weak, irregular shape (blue-green on heatmap, not rectangular) → DTD noise.
I. In margins / blank borders, far from content, visually empty → margin false positive.
J. Math/formula in math-heavy document with massive DTD activation, formula renders cleanly → DTD misfiring on math.

If a region matches none of H/I/J → KEEP IT.

==================================================================
TYPE CLASSIFICATION
==================================================================

* Visual Clumsy — visible editing artefact: blur, pixelation, redaction block, overpainting.
* Logical Fraud — specific value (number, date, statistic, ID, year, name) altered creating factual inconsistency. Digits may render fine; evidence is the VALUE being wrong vs document's own logic.
* Semantic Subtle — clean word substitution creating semantic contradiction with surrounding text.

Rules:
* A DTD region is NOT automatically Visual Clumsy — read the content.
* Use Semantic Subtle when text is visually clean but contextually wrong.
* Use Logical Fraud when a value is numerically/factually wrong.
