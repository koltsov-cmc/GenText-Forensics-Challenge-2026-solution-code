=========
ABOUT DTD — TRUST IT, BUT VERIFY THE TYPE
=========

DTD is a strong, specialised visual-tampering detector. On test data it achieves 0.66 IoU against ground-truth tampering masks — meaning when DTD flags a region, that region is GENERALLY a real tampered area, not noise. Treat DTD's flagged regions as **strong prior evidence of tampering**, not as suggestions you can casually discard.

DTD primarily looks for STATISTICAL DEVIATIONS in the document: font mismatches, edge artefacts, redaction blocks, JPEG-compression discontinuities, kerning anomalies, baseline shifts, rectangle filling, white-out areas, blurred regions, and similar low-level cues.

However, DTD has three important blind spots:

* **DTD does NOT classify the type of tampering.** A flagged region might be a crude visual redaction (Visual Clumsy), an altered numeric value (Logical Fraud), or a clean word substitution that creates a semantic contradiction (Semantic Subtle). YOU must read the actual content of the region and classify it.

* **DTD misses pure-semantic substitutions.** When a forger replaces a clean word with another clean word that introduces no visual artefact (e.g. "Preview" → "Review", "Save" → "Share"), DTD often misses it entirely. Finding these is YOUR additional task — they are not in the DTD list but they exist in the document.

* **DTD has known failure modes.** It triggers heavily on mathematical formulas, equations, and dense numeric tables, and on logos/stamps/seals. The drop rules below tell you when to suspect a DTD failure mode.

=========
WHEN TO KEEP A DTD REGION (default = keep)
=========

Given DTD's 0.66 IoU baseline, the default is KEEP. Drop a region only when one of the explicit drop rules below applies. Most of your work is deciding the TYPE, not whether to keep the region.

CASES STRONGLY FAVOURING KEEP:

A. **Region inside a paragraph or between words.** → most often **Visual Clumsy** if the artefact is visible, **Semantic Subtle** if the substituted word is visually clean but contextually wrong, **Logical Fraud** if a numeric value was changed.

B. **Blurred / JPEG-distorted text** → **Visual Clumsy**.

C. **Solid rectangle / white-out / black-out** → **Visual Clumsy**.

D. **Empty area in the middle of running text** → **Visual Clumsy** or **Logical Fraud** (removed value).

E. **Blurred logo / stamp / seal** while surrounding document is sharp → **Visual Clumsy**.

F. **Numeric value where the digits look clean** (tightly-bounded number, date, percentage, total, page number, statistic) → **Logical Fraud**.

G. **Single word or short phrase, visually clean, in running prose that contradicts surrounding context** → **Semantic Subtle**.

=========
WHEN TO DROP A DTD REGION (rare; only the cases below)
=========

H. **Tiny, weak, irregular shape**, mostly blue-green on the heatmap rather than red/yellow, AND not approximately rectangular. → DTD noise, drop.

I. **In document margins / blank borders, far from any content,** AND visually empty. → low-quality false positive, drop.

J. **Math content in a math-heavy document with massive DTD activation,** AND the formula renders cleanly. → DTD misfiring on math glyph statistics, drop.

If a DTD region does not match any of H/I/J — KEEP IT. The penalty for dropping a true positive is severe.

=========
TYPE CLASSIFICATION
=========

Every kept region must be assigned EXACTLY ONE of three types:

* **Visual Clumsy** — visually obvious manipulation: blur, pixelation, smearing, hard-edge redaction blocks, visible overpainting, copied blocks, clear obscuration.

* **Logical Fraud** — a specific value (number, date, statistic, page number, year, count, name, ID) has been altered, creating factual inconsistency. The digits may render fine; the evidence is the value being wrong relative to the document's own logic.

* **Semantic Subtle** — a word or short phrase was replaced with a visually clean substitution that creates a semantic contradiction with surrounding text.

CRITICAL CLASSIFICATION RULES:

* A DTD-flagged region is NOT automatically Visual Clumsy. Read the content, do not classify by source.
* A rectangular shape, black bar, or white box is NOT by itself enough to call something Visual Clumsy — it must show clear visual obscuration or editing artefacts.
* Use Semantic Subtle when the text is visually clean but contextually wrong.
* Use Logical Fraud when a value is numerically or factually inconsistent.

=========
ABOUT OCR
=========

OCR is GROUND TRUTH for two things only:
* knowing approximately what text appears where in the image,
* mapping each kept region to the words that overlap it.

OCR confidence is **NOT a tampering signal**. Do NOT keep/drop a region based on OCR confidence.

OCR errors (e.g. "StHl" instead of "STAY") are NOT tampering. Only flag what you can verify VISUALLY in the image.

=========

Do NOT add anomalies "just to be safe". Each anomaly must be a CONCRETE, NAMEABLE substitution where you can articulate both the tampered version and the likely original version. A small number of high-quality anomalies is far better than a long list with false positives.

The anti-padding rule does NOT mean "drop DTD regions you are unsure about" — DTD regions are kept by default and only dropped via H/I/J.
