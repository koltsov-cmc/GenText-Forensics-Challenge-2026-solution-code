==================================================================
HOW TO CLASSIFY EACH DTD REGION
==================================================================

For each DTD region, look at the image and at the OCR text overlapping the region. Assign exactly one of three types:

* **Visual Clumsy** — visible editing artefact in the image: blur, pixelation, smearing, hard-edge redaction blocks, solid rectangles, white-out / black-out, overpainting, JPEG halo, visible obscuration.

* **Logical Fraud** — the region contains a specific value (number, date, statistic, page number, year, count, ID, name, amount) that renders cleanly but is likely altered relative to what it should logically be.

* **Semantic Subtle** — the region contains a clean word or short phrase whose meaning contradicts the surrounding sentence (e.g. "for-profit charity", "permanent temporary", a wrong product name in a technical list).

Helpful guidance for picking the type:

A. Region inside a paragraph or between words → likely **Visual Clumsy** if there is a visible artefact, **Semantic Subtle** if the substituted word is visually clean but contextually wrong, **Logical Fraud** if a numeric value was changed.
B. Blurred / JPEG-distorted text → **Visual Clumsy**.
C. Solid rectangle / white-out / black-out → **Visual Clumsy**.
D. Empty area in the middle of running text → **Visual Clumsy** or **Logical Fraud** (removed value).
E. Blurred logo / stamp / seal while surrounding document is sharp → **Visual Clumsy**.
F. Numeric value with clean digits (number, date, percentage, total, page number, statistic) → **Logical Fraud**.
G. Single visually-clean word or short phrase in running prose that contradicts surrounding context → **Semantic Subtle**.

Tie-breaker if unsure:
* Region contains DIGITS → Logical Fraud
* Region contains LETTERS only → Semantic Subtle
* Region empty / blank / obviously redacted → Visual Clumsy

CRITICAL CLASSIFICATION RULES:

* A DTD-flagged region is NOT automatically Visual Clumsy — read the content and pick the right type.
* A rectangular shape, black bar, or white box is NOT by itself enough to call something Visual Clumsy — it must show clear visual obscuration or editing artefacts.
* Use Semantic Subtle when text is visually clean but contextually wrong.
* Use Logical Fraud when a value is numerically or factually inconsistent.
